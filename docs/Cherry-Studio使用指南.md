# Cherry Studio 使用指南

## 为什么推荐 Cherry Studio

1. **Agent 工作目录功能**：可以让 AI 直接访问整个项目文件夹，无需手动上传知识库
2. **多模型对比**：可以用不同模型（Claude/GPT/Gemini）测试章纲效果
3. **MCP 服务器支持**：可选配置 NovelForge 的 MCP 服务器获得增强功能
4. **会话管理**：每章创作可以独立会话，方便回溯和对比

## 推荐配置方式

### 方式一：Agent + 工作目录（推荐）

**适合**：使用支持 Agent 功能的模型（Claude、Gemini 等）

**步骤**：

1. **创建专用 Agent**
   - 在 Cherry Studio 中创建新 Agent
   - 名称：`NovelForge 创作助手`
   - 系统提示词：复制 `docs/CherryStudio初始化prompt.md` 中分隔线内的内容
   - **设置工作目录**：选择你的小说项目根目录（包含 `CLAUDE.md`、`知识库/`、`工作区/` 的那个文件夹）

2. **开始创作**
   - 新建对话，选择刚创建的 Agent
   - AI 会自动读取 `CLAUDE.md` 和知识库文件
   - 按 6 阶段工作流进行（章纲发散 → 人类选择 → 正文产出 → AI 自查 → 问题修正 → 终版归档）

**优点**：
- AI 可以直接访问所有知识库文件，无需手动选择
- 自动读取最新的 `当前状态.md` 和 `最近章节梗概`
- 可以直接在 `工作区/` 创建和修改文件

**注意**：
- 确保工作目录路径正确
- 首次对话时 AI 会自动初始化（读取 CLAUDE.md 和核心上下文）

---

### 方式二：知识库导入（备用）

**适合**：使用不支持 Agent 功能的模型（某些 GPT API、Codex 等）

**步骤**：

1. **导入知识库**
   - 使用 Cherry Studio 的"知识库"功能
   - 将 `知识库/` 目录下的所有 `.md` 文件上传
   - 将 `CLAUDE.md` 也上传

2. **每次对话手动选择文件**
   - 新建对话时，从知识库中选择需要的文件
   - 至少选择：`CLAUDE.md`、`当前状态.md`、`最近章节梗概`

**缺点**：
- 需要手动选择文件，容易遗漏
- 无法直接在 `工作区/` 创建文件
- 每次更新知识库需要重新上传

## 可选：配置 MCP 服务器

如果你想使用 NovelForge 的 MCP 服务器（提供知识库查询、文件操作等增强功能）：

1. **启动 MCP 服务器**
   ```bash
   cd novel-knowledge-mcp-server
   npm run build
   npm start
   ```

2. **在 Cherry Studio 中配置 MCP**
   - 打开 Cherry Studio 设置
   - 找到 MCP 服务器配置
   - 添加本地服务器地址（通常是 `http://localhost:3000`）

3. **可用工具**
   - `read_knowledge_file` - 读取知识库文件
   - `list_knowledge_files` - 列出知识库文件
   - `search_knowledge` - 搜索知识库内容
   - `get_current_status` - 获取当前创作状态

**注意**：MCP 服务器是可选的，使用 Agent 工作目录功能已经足够完成所有创作任务。

## 工作流示例

### 创作新章节（第 28 章）

1. **新建对话**，选择 `NovelForge 创作助手` Agent

2. **发起请求**：
   ```
   开始第 28 章的章纲发散
   ```

3. **AI 自动初始化**：
   - 读取 `CLAUDE.md`
   - 读取 `知识库/00_核心上下文/当前状态.md`
   - 读取 `知识库/00_核心上下文/最近章节梗概（第21-27章）.md`

4. **AI 生成 3 版章纲**，你选择一版或提出修改

5. **AI 写正文**：
   - 自动读取相关人物档案
   - 自动读取场景范例
   - 自动读取微调意图学习库

6. **AI 自查**：
   - 自动读取错题集、去重表
   - 对照终版自查清单
   - 输出修正后的正文 + 自查报告

7. **你反馈修改** → AI 修订 → 确认终版

8. **AI 更新知识库**：
   - 更新 `当前状态.md`
   - 更新 `最近章节梗概`
   - 更新 `伏笔追踪表`
   - 更新 `微调意图学习库`

## 常见问题

**Q: Agent 工作目录和知识库导入有什么区别？**

A: Agent 工作目录让 AI 可以直接访问整个项目文件夹，就像在本地文件系统中工作一样。知识库导入是将文件上传到 Cherry Studio 的知识库中，每次对话需要手动选择文件。前者更方便，后者适合不支持 Agent 功能的模型。

**Q: 我需要配置 MCP 服务器吗？**

A: 不需要。使用 Agent 工作目录功能已经足够完成所有创作任务。MCP 服务器只是提供了一些额外的便利工具（如知识库搜索），但不是必需的。

**Q: 每次新会话都要重新初始化吗？**

A: 是的。每次新会话开始时，AI 会自动读取 `CLAUDE.md` 和核心上下文文件，确保使用最新的状态和规则。这个过程是自动的，你不需要手动操作。

**Q: 我可以在同一个会话中创作多章吗？**

A: 可以，但不推荐。建议每章创作使用独立会话，这样可以：
- 避免上下文混乱
- 方便回溯和对比不同章节的创作过程
- 减少 token 消耗

## 个人使用建议

- **章纲阶段**：可以尝试多个模型对比（Claude Opus 4.6 擅长结构，Sonnet 4.6 更快）
- **正文阶段**：推荐使用 Claude Opus 4.6，文风把控更好
- **自查阶段**：AI 会自动完成，你只需要审阅自查报告
- **微调阶段**：你的每次修改都会被 AI 学习，后续章节会自动改进
